This page lists and briefly describes all the main documents, of the EPICS V4 Working Group.

The documentation includes programmers references, Normative track documents of the Working Group, Internal documents, Alpha level software documentation, proposal documents, talks and papers.

Normative Track Documents

EPICS V4 is a body of core specifications, reference implementations, and some associated tools. The core specifications, we call "normative". The normative specifications are open, collaboratively defined, and may be implemented and extended by any interested party. We provide reference implementations for adopters of EPICS V4, and to test the specifications' performance and effectiveness. Associated other documents are "non-normative" and no guarantees will be made about them.

  • pvAccess Protocol Specification. This document formally describes the network communications protocol of EPICS V4 endpoints, in sufficient detail that it can be independently implemented and extended. Status is Second Public Working Draft
  • EPICS V4 Overview and Architectures. This document summarises the function of various technologies in the EPICS V4 sphere, and outlines how they might be used together in various design patterns for efficient control systems. Status is Editors Draft
  • Normative Types Specification. The EPICS V4 Normative Types are a collection of data types that may be used by the application level of EPICS V4 network endpoints, to interoperably exchange scientific data. Status is Editors Draft

Informative

The following are works of the EPICS V4 Working Group that are intended to supply supporting material. They may be concerned with tools and utilities supplied by EPICS V4, but they should not be considered standards or recommendations for EPICS Version 4:

  • Getting Started with EPICS V4. This is a first steps introduction to downloading EPICS V4, writing your first data server, and connecting to Base IOCs through pvAccess and writing RPC servers. Java and C++ steps are given. Status is Complete and up to date with respect to V4.4
  • EPICS V4 Developer's Guide. This document briefly describes the application programming interfaces (API) for a number of components of EPICS V4. Status is Ongoing Draft
  • Overview of EPICS version 3 and EPICS version 4. This document was a comparison of EPICS V3 and V4 as they existed prior to EPICS V4.3. It also describes the straw man V4 IOC framework called pvIOC in comparison to the in V3. Status is describes deprecated software, though useful as a reference for future work.
  • PSI EPICS Installation Example. This is a PSI internal design document for how EPICS V4 Java support for services, and actual first data services themselves (for accessing an Oracle DB), were deployed at PSI. Status is Working Draft - this is an "as implemented" description.
  • Troubleshooting EPICS V4. This document is a short list of the very first common mistakes and problems that may be encountered when starting out with developing EPICS V4, together with suggested fixes. Status is Ongoing Draft of an Internal document - expect maintenance on a best effort basis only.
  • pvRequest The Channel interface has a set of create methods that have an argument named pvRequest. This document discusses how this argument is defined and used. Status is Editors draft.
  • pvAccessSemantics This document discusses the semantics for pvAccess. Status is Editors draft.

Internal, Proposals, Alpha Software Documentation, and Early Drafts

The following are internal documents of the EPICS v4 Working Group:

  • Charter of the EPICS V4 Working Group. Charter is a statement of the basis for the work of the of the EPICS V4 working group, the intended outcomes for that work, its deliverables and success criteria
  • EPICS Version 4 Process. The Process document describes the administrative setup of the EPICS Version 4 working group effort, and defines workflow. In particular, it defines the specification draft publication stages. Status is completed.
  • Feature List Release 4.4. This is a list of new features EPICSVersion 4 Release 4.4.
    Proposed Changes To Release 4.3.0. A brief description of the changes to be made to the modules that are part of EPICSVersion 4 Release 4.3.0. Status is Working.
  • Coding Style Guidelines. This is the standard for source code to be followed for EPICS V4 reference software implementations. Status is it includes only Java, and requires some edits with respect to comments in the review telecon
  • Unsigned Integer data exchange support - Proposal. The design proposal for how direct support of unsigned integers can be added to pvAccess and pvData. Status is Working Draft, this draft forms the basis for the unsigned support being added to EPICS V4 (Spring 2012)
  • Easy PvAccess (easyPVA). Programmers guide to a simple API for exchanging pvData defined data across the pvAccess protocol. Status of the documentation is Editors Draft. JavaDoc; status of the software is Usable Alpha. easyPVAJava module in SF. See also Easy API - Proposal below
  • Easy API - Proposal. This is a one-page description of a wrapper for the above easyPVA EPICS V4 API, intended for truly trivial EPICS V4 client programming. Status: Simple proposal document, intended to illustrate functionality to be integrated into easyPVA
  • Provider - Proposal. Description of how a client determines the provider to be used when connecting to a channel.
  • Gateway Requirements Document. Collecting requirements for the next generation EPICS Gateway module. Status: Working draft, to be finalized at Summer 2014 WG meeting.
  • Gather service platform. The Gather Service Platform is a software toolkit on top of EPICS V4 pv modules, to help build a service for efficiently managing the acquisition of the values of a large number of V3 PVs, and making results available as an EPICS V4 record. Gather Service Status can be used as a fledgling programmers reference document. Gather Examples and Tests. Status: These documents are early maturity documents towards programmers documentation.   • Cheatsheet for Passwordless SSH Access to SourceForge for EPICS V4 Development. Status: up to date.
  • Access to the EPICS V4 Webspace on SourceForge. Status: up to date.
  • Doxygen C++ source code documentation generation cheatsheet for EPICS V4 development. Status: up to date.
  • EPICS V4 Branched Development with Mercurial (hg flow). Status: up to date.
  • EPICS V4 Versioned Release Guide. Status: up to date.
